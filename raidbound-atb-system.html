Raidbound ATB System

Charge-based pacing designed to keep menu combat active and readable

A timing layer built for clarity, not chaos

← Back to Home

Raidbound uses an Active Time Battle structure to keep fights moving while preserving tactical control. Instead of long static turns, the player responds as heroes reach readiness. That creates urgency, but the interface still keeps decisions understandable.

  • Charge-based readiness
  • No spell queue clutter
  • Hero-by-hero decisions
  • Visual readiness feedback

Design Problem

A lot of fast combat systems become hard to read once several characters, effects, and abilities are moving at once. I wanted Raidbound to feel active, but I did not want the player wrestling with a timeline they could barely parse.

The answer was a controlled ATB layer: enough motion to keep battles alive, but enough structure that action availability, role identity, and effect timing can still be understood quickly.

How It Works

Build charge

Heroes accumulate toward an action threshold over time, creating a steady rhythm of readiness instead of all-or-nothing turns.

Reach full state

When a hero is ready, the UI presents available actions clearly. Full readiness is emphasized visually so the player can spot it instantly.

Spend through actions

Attacks and abilities consume readiness according to their design, allowing pacing differences between simple actions and stronger options.

Return to flow

Once resolved, the hero re-enters the shared pace of battle and starts building toward the next decision point.

Why It Fits Raidbound

It supports party play. Multiple heroes can come online at different times, which makes role coverage and party planning more important.
It rewards UI clarity. Gauge glow, card state, and action visibility become meaningful parts of gameplay communication.
It scales well with systems. Items, buffs, debuffs, and encounter types can influence battle pacing without requiring a completely different combat framework.

Technical Value

Balance friendly

A deterministic timing model is easier to simulate, test, and tune than loosely defined action chaos.

Portfolio signal

This system shows combat pacing design, UI communication, and technical thinking all in one feature.

Future tooling

The ATB layer is a strong candidate for simulators, visualizers, and spreadsheet-backed balance workflows.
