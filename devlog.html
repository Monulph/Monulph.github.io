March 2026

Raidbound Web Demo Live

I finished a dedicated web demo version of Raidbound and got it hosted through GitHub Pages. Instead of trying to force the full game into a browser build, I created a tighter public-facing slice that loads faster, avoids unfinished systems, and gives players a cleaner first impression.

Demo Build Changes

  • Removed audio for the web demo and made sure the game remained stable without it
  • Removed the tutorial, profile selection flow, and options menu from the public build
  • Auto-assigned players to Profile 1 to simplify startup
  • Trimmed the hero pool down to Bulwark and Luminary only
  • Locked the available item pool to the first 12 default items only
  • Removed elite nodes, rare nodes, quest item flow, and all of Act 2
  • Shortened Act 1 into a compact 3-row run with 2 normal fights and 1 boss
  • Routed the Act 1 boss victory straight into the existing thank-you-for-playing demo screen
  • Removed boss loot and Act 2 reinforcement flow from the demo path

Web Release Cleanup

  • Tracked down missing-art and missing-background issues caused by cutting unused content
  • Cleaned up combat background references so the trimmed project no longer throws missing resource errors
  • Removed item unlock popups since hero and item collection screens are not part of the web demo
  • Added a Play Demo link to the Raidbound section of the portfolio
  • Moved larger site/demo updates into GitHub Desktop so the exported web files could be pushed properly

Next focus: Continue polishing the hosted demo experience and keep the portfolio aligned with the current state of Raidbound.
