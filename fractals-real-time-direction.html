Fractals Real-Time Direction

Automation and control working together to shape a continuous visual session

A direction layer for choosing when the system changes and when it holds still

← Back to Home

Fractals is not meant to be a fixed slideshow of effects. It is a directed real-time system that chooses when to transition, how long to stay with a scene, and when the user should be able to hold onto a moment. The project balances automated evolution with hands-on control so it can work for passive viewing, live experimentation, and captured recordings.

  • Automated scene flow
  • User scene locking
  • Transition control
  • Live creative direction

Design Problem

A visualizer loses impact if it changes too often, but it also becomes flat if it never evolves. The hard part is not just making good visuals. It is deciding when the system should move forward, when it should hold, and how to avoid transitions that feel abrupt or artificial.

Fractals needed a direction layer that could preserve flow while still giving the user ways to control what stays on screen, especially for recording, showcasing, or studying particular scenes.

How It Works

Choose active scene

The system determines which scene or scene combination is currently driving the visual output.

Control transition timing

Scene changes are directed to avoid hard cuts that would break the mood or make the visual session feel mechanical.

Allow user intervention

Controls like scene locking make it possible to hold a visual moment instead of always surrendering to automation.

Support layered playback

As the project evolves, direction systems can also support running multiple scenes together for denser visual behavior.

Why It Fits Fractals

It respects pacing. The project works best when visual changes feel earned and rhythmic rather than random or timer-driven.
It supports creators. Recording or showcasing the project becomes easier when the user can lock scenes and guide what stays visible.
It pushes the system beyond a demo. Direction logic turns Fractals into a real-time experience with intent, not just a rotating list of effects.

Technical Value

Behavioral design

This work connects graphics to state logic, timing control, and interaction design rather than leaving them as separate concerns.

Portfolio signal

It shows systems thinking about how procedural content is presented over time, not only how it looks in a still frame.

Creative utility

Control features make the project more useful in practice, especially for recording, curation, and future live presentation.
