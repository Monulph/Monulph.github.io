What it is

Fractals is an interactive visualizer project built as both a standalone visual experience and a larger technical art system. It explores how shader-driven visuals can respond to sound, evolve over time, and eventually support more than pure presentation alone.

The long-term goal is not just to make interesting screens, but to build a reusable visual language that can apply to music playback, live interaction, recording, and future game-facing presentation systems.

Photosensitivity warning: This fractal experience includes flashing and strobe-like visual effects. Please proceed with caution if you are sensitive to rapidly changing light or patterns.

Why it matters

Fractals sits at the intersection of technical art, rendering, and visual system design. It is a place to explore shaders, procedural motion, scene transitions, deployment, and performance while still producing something that feels artistic and experiential.

That makes it more than a visual toy. It is also research and development for future projects where the screen itself can carry mood, feedback, pacing, and identity.

How I think about the project

  1. Build scenes that feel visually distinct but still belong to one coherent system.
  2. Use shaders and real-time rendering as the core visual engine.
  3. Improve transitions, responsiveness, and load behavior so the project feels alive.
  4. Keep the project deployable in both browser and desktop environments.

The best version of Fractals is not just a cool effect. It is a stable, scalable, and well-presented graphics project that demonstrates both artistic direction and systems thinking.
