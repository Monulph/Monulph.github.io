Overview

Fractals is a real-time visual project built to explore shader-driven motion, scene flow, rendering behavior, and technical art direction in a live interactive format.

What began as a visualizer has grown into a larger portfolio system focused on how procedural visuals can be shaped, organized, optimized, and eventually applied to broader game-facing presentation.

  • Shader-driven scenes
  • Real-time visual flow
  • Scene and family architecture
  • Browser and desktop builds
  • Technical art portfolio project

What makes this project important

Fractals sits at the intersection of technical art, rendering, visual design, and systems thinking. It is a place to explore how shaders, motion language, timing, and performance can work together to create something that feels both artistic and structured.

The goal is not just to generate interesting screens. The goal is to build a reusable visual system that can eventually inform future game presentation, menu motion, environmental energy, transitions, and moment-to-moment visual feedback.

Project highlights

Real-time rendering

Built around continuously updating visuals rather than static effects, with an emphasis on flow, motion, and screen presence.

Scalable structure

Scenes are being organized into families and systems so the project can grow intentionally instead of becoming a collection of disconnected experiments.

Technical art framing

This is meant to show more than visuals alone. It demonstrates rendering decisions, system design, iteration, deployment, and optimization.

Game-facing potential

The long-term value of Fractals is in how these ideas may influence future game feel, visual pacing, and reactive presentation systems.

How I think about the project

  1. Build scenes that feel visually distinct while still belonging to one larger visual language.
  2. Use shaders and rendering logic as the foundation instead of treating visuals as decoration added afterward.
  3. Improve transitions, pacing, responsiveness, and startup behavior so the project feels alive and intentional.
  4. Keep the project presentable in both browser and desktop environments.
  5. Document the systems clearly enough that the project reads as technical art work, not just a cool demo.

The strongest version of Fractals is a polished, scalable graphics project that demonstrates artistic direction, rendering curiosity, and practical systems thinking.

Why do I keep working on Fractals?

Fractals is one of the clearest examples of how I approach technical art problems: start with a visual target, build the underlying system, improve how it performs, and shape it into something presentable and expandable.

Instead of stopping at isolated shader studies, this project pushes toward a more complete portfolio piece with architecture, deployment, optimization, direction systems, and documented breakdown pages.
