Fractals is built to live in more than one environment. It runs in the browser through GitHub Pages and also as a desktop application through Electron. That dual deployment path matters because it changes who can access the work, how it performs, and how the project can be presented as both a portfolio piece and a standalone creative tool.

  • Browser deployment
  • Electron desktop build
  • Cross-environment rendering
  • Shareable technical art project

Design Problem

A project like Fractals gains value when people can actually access it, but graphics-heavy work often behaves differently across environments. Browser rendering, desktop packaging, startup flow, and shader handling can introduce different problems even when the visuals are built from the same core code.

The project needed a deployment path that could support both easy public access and a stronger desktop presentation without splitting into two unrelated versions.

How It Works

Build for web

The browser version allows the project to live publicly as part of the Monulph portfolio and makes the work immediately accessible.

Build for desktop

The Electron version supports a standalone experience and creates room for more controlled presentation on local machines.

Resolve environment issues

Renderer setup, shader compilation, and load behavior are reviewed across both paths so the project remains dependable.

Present as a system

Deployment is treated as part of the project design because accessibility and execution quality both shape how the work is perceived.

Why It Fits Fractals

It improves reach. The web version makes the project easy to share, which is critical for portfolio visibility and public experimentation.
It supports a stronger showcase. The desktop path creates room for a more controlled environment when browser limitations are not ideal.
It reflects production reality. Technical art work often has to survive different runtimes, not just the one where it was first authored.

Technical Value

Platform awareness

This work shows an understanding that rendering, setup, and presentation shift across environments and must be handled intentionally.

Portfolio signal

It demonstrates that the project is not only visually interesting but also packaged and deployed in ways that make it usable and reviewable.

Long-term flexibility

Maintaining both deployment paths keeps the project adaptable for sharing, iteration, and future expansion.
